Instruction Life Cycle

The instruction lifecycle refers to the sequence of stages a CPU follows to execute an instruction. Here’s a step-by-step explanation:

    Fetch Operand
  1. The Program Counter (PC) points to the address of the next instruction.
  2. The instruction is fetched from memory and placed in the Memory Buffer Register (MBR).
  3. The Instruction Register (IR) stores the fetched instruction.
  4. The PC is incremented to point to the next instruction.
    Decode
  1. The instruction in the IR is decoded by the Control Unit (CU).
  2. The CU determines what action needs to be performed (e.g., arithmetic operation, memory access, jump).
  3. It identifies the operands and any necessary data for the operation.
    Fetch Operad
  1. If the instruction requires operands (data), the addresses of the operands are sent to the Memory Address Register (MAR) to fetch them from memory.
  2. The operands are loaded into the appropriate registers or memory.
    Execute
  1. The Arithmetic Logic Unit (ALU) performs the specified operation (e.g., addition, subtraction, logical operations).
  2. The result of the operation is generated.
    Write-Back
  1. The result from the ALU is written back to the destination register or memory location, as needed.
